# Chaos League – Project Documentation

This folder contains the AI context and project documentation for Chaos League.

These files provide system understanding for AI agents such as Codex and Cursor before implementing changes.

Agents must read this documentation before performing development tasks.

## Purpose

Provide structured project context including:

- system architecture
- product requirements
- development constraints
- UX expectations
- active tasks
- system decisions
- persistent project memory

## How Agents Should Use This

Before implementing changes, agents should review:

1. ai-context/project-overview.md
2. ai-context/architecture.md
3. ai-context/constraints.md
4. product/product-requirements.md
5. docs/system-map.md
6. tasks/current-sprint.md

Agents should then summarize understanding before coding.

## Key Principle

Documentation first.  
Code second.
